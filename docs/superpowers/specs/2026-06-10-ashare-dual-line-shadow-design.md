# 个人 A股 多因子「左右双线」影子盘系统 — 设计文档

- 日期：2026-06-10
- 状态：已与用户确认，待写实现计划
- 语言/技术栈：Python（pandas/numpy/akshare/pyarrow/sqlite/matplotlib）

## 1. 目标与定位

一个**个人 A股 量化研究系统**，借鉴机构的决策纪律（假设驱动、严格回测、风控、影子盘前向验证），但裁剪到个人体量：

- **频率**：日频。每个交易日收盘后跑一遍，出信号。
- **使用方式**：系统出信号，用户**手动下单**（不接券商自动交易）。
- **左右双线**：同时运行两套相反逻辑的策略，各自一个独立影子账户。
  - **左侧**：逆势 / 价值 / 均值回归（抄底）。
  - **右侧**：顺势 / 动量 / 趋势（追涨）。
- **影子模式**：两个账户按 A股 **真实交易规则模拟成交**，不投真钱，持续累积样本外虚拟业绩，作为"该不该真的下单"的客观依据。
- **代码复用**：回测与影子盘复用**同一套策略 + 撮合引擎代码**，保证回测结论与实盘一致。

## 2. 账户设定

| 账户 | 本金 | 策略 | 选股榜 |
|---|---|---|---|
| 左侧账户 | 20 万 | 逆势/价值 | 左侧因子榜 |
| 右侧账户 | 20 万 | 顺势/动量 | 右侧因子榜 |

- 两账户**完全独立**：各自现金、持仓、记账、盈亏。
- **同一只股票允许被两边各自持有**（左右因子逻辑相反，重叠极少，但不做跨账户去重）。

## 3. 数据层

- **数据源**：AKShare（免费，全市场量价 + 财务）。抽象 `DataSource` 接口，未来可替换为 Tushare 而不改上层。
- **本地缓存**：行情用 Parquet，元数据/账户/交易流水用 SQLite。每日收盘后增量更新当天数据后 append。回测全部读本地缓存，不依赖每次联网。
- **复权**：行情取**前复权**。
- **交易日历**：接入 A股 交易日历，节假日休市跳过；只在交易日运行与成交。

## 4. 选股池（左右共用同一个基础池）

每个交易日动态构建：

- 范围：全 A股 **＋ 科创板（688）**。
- 剔除：ST/\*ST、当日停牌、上市 < 60 个交易日、北交所。
- 流动性过滤：剔除近 20 日日均成交额过低的标的（阈值可配置）。

## 5. 因子库（初始版本，机构常见默认值，回测后可调）

每个因子：去极值 + 标准化 → 等权合成 → 排名取前 N。因子在 `config.yaml` 中可配置。

- **左侧因子族（逆势/抄底）**：
  - 低估值（PB / PE 低分位）
  - 超跌乖离（距 MA20 / MA60 负乖离）
  - RSI 超卖
  - 近 20 日跌幅
- **右侧因子族（顺势/追涨）**：
  - 动量（20 / 60 日涨幅）
  - 均线多头排列
  - 创 60 日新高
  - 量价齐升

## 6. 每日选股流程

1. 构建当天基础池（第 4 节）。
2. 同一池子用**两套因子各打一次分**，得到左侧榜与右侧榜。
3. 各取榜单前 N，过滤掉当日不可交易的（涨停一字板买不进、停牌剔除）。
4. 结合各自账户状态决定动作：
   - 有空仓位（持仓 < 目标只数）→ 从本侧榜前列挑「未持有」的买入。
   - 已持有 → 逐只判断加仓 / 卖出。
5. 按目标仓位算股数（÷价格，向下取整到整手；科创板 ≥200 股且以 1 股递增；现金不足则跳过）。
6. 输出今日操作清单。

## 7. 策略规则

### 7.1 建仓与仓位
- 每账户目标持有 **3–5 只**，等权目标仓位。
- 金字塔**分批建仓**（非一次买满）。

### 7.2 加仓
- **右侧（正金字塔）**：浮盈 +5% 或突破前高时加一笔，**越加越少**（首仓 50% → 30% → 20%），总仓不超目标。加仓份额受 T+1 锁定。
- **左侧**：**不做向下摊薄加仓**（避免马丁格尔）。首仓买够，或反弹站上 MA5 确认后再加。

### 7.3 卖出
- **右侧**：`收盘价 < MA10` → 当日尾盘（以收盘价）卖出。
  - 说明：用 15:00 收盘价代理用户的「14:59 尾盘价」。残留误差仅限刚好贴 MA10 的临界股（收盘集合竞价 3 分钟波动），可接受；要精确需分钟数据，对日频不值得。
- **左侧**（任一触发即卖）：
  - 止盈：反弹站上 MA20，或浮盈 +15%
  - 因子止盈：因子打分掉出前 N
  - 硬止损：跌破建仓价 -8%
  - 时间止损：持有满 20 个交易日仍未反弹

### 7.4 持有天数
- 持有天数按**首次建仓**计，中途加仓**不重置**。

## 8. 影子撮合引擎（核心 · 真实模拟）

两个独立账户对象，各自维护现金、持仓、持有天数、分批建仓状态。逐笔套用 A股 真实规则：

- **T+1**：当日买入（含加仓）份额当日锁定，不可卖；昨日及以前的「活仓」当日可卖。
- **交易时段**：只在交易日成交，节假日跳过。
- **费用**：
  - 佣金：成交额 × 万 0.85（0.0085%），**最低 5 元/笔**
  - 印花税：卖出单边 0.05%
  - 过户费：双边 0.001%（万 0.1）
  - 滑点：0（可配置固定 bp）
- **板块差异规则**（按 `board` 区分）：
  - 涨跌停幅度：主板 ±10%、创业板(300)/科创板(688) ±20%、ST/\*ST ±5%
  - 涨停（一字/封板）→ 买不进；跌停 → 卖不出
  - 停牌 → 当日不可交易
  - 整手：主板/创业板 100 股整数倍；科创板 ≥200 股且以 1 股递增
- **撮合价**：卖出按当日收盘价（尾盘规则）；建仓价默认收盘价，可配置开盘价。

## 9. 回测引擎

- 事件驱动，按交易日循环：每日 → 更新行情 → 算因子/信号 → 过撮合引擎 → 更新账户 → 记录净值。
- 输出指标：净值曲线、年化收益、夏普、最大回撤、胜率、换手率，并按左/右账户分别归因。

## 10. 每日影子运行（runner）

- 每日收盘后跑一次：更新数据 → 生成今日信号 → 影子账户记账 → 输出：
  - **今日操作清单**（左/右分别：买入清单含目标股数与价、加仓、卖出）
  - 当前持仓状态
  - 当日及累计虚拟盈亏
- 与回测复用同一套策略 + 撮合代码。

## 11. 报告/输出层

- 每日信号报告：markdown / 控制台 / CSV。
- 回测报告：指标表 + 净值图（matplotlib）。

## 13. 技术取舍

- **不用 backtrader/zipline/qlib 等现成框架**：A股 的 T+1、板块涨跌停、双账户、金字塔加仓等定制规则塞进现成框架反而别扭；自建轻量引擎更可控、更贴合"真实模拟"，也是机构自研回测的常见理由。
- 抽象数据源接口，便于未来从 AKShare 升级到 Tushare。
- 回测与影子盘共用策略 + 撮合代码，杜绝"回测一套、实盘一套"的偏差。

## 14. 暂不做（YAGNI）

- 不接券商自动交易（手动下单）。
- 不接分钟/分时/Level-2 数据。
- 不做高频、另类数据、组合优化器（个人体量用等权 + 简单规则即可）。
